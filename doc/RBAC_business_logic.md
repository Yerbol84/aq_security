⏺ 📘 RBAC Business Logic - Полная бизнес-логика системы                                                                                                                                                                                                                                           
                                                                                   
  1. User Behavior - Поведение пользователей                                                                                                                                                                                                                                                      
                                                                                                                                                                                                                                                                                                  
  1.1 Сценарии взаимодействия                                                                                                                                                                                                                                                                     
                                                                                                                                                                                                                                                                                                  
  Администратор системы:                                                                                                                                                                                                                                                                          
  - Создает роли (Admin, Editor, Viewer, ProjectManager)                                                                                                                                                                                                                                          
  - Настраивает иерархию ролей (Editor наследует от Viewer)                                                                                                                                                                                                                                       
  - Назначает права ролям (projects:create:own, files:read:*)                                                                                                                                                                                                                                     
  - Создает политики доступа (запрет доступа вне рабочих часов)                    
  - Назначает роли пользователям                                                                                                                                                                                                                                                                  
  - Выдает временные роли (на 24 часа для инцидента)                                                                    
  - Мониторит логи доступа и оповещения                                                                                                                                                                                                                                                           
                                                                
  Обычный пользователь:                                                                                                                                                                                                                                                                           
  - Логинится в систему (получает JWT токен с userId)                              
  - Пытается выполнить действие (открыть проект, удалить файл)                                                                                                                                                                                                                                    
  - Система автоматически проверяет права                                          
  - Видит UI элементы согласно своим правам (Guard widgets скрывают недоступное)                                                                                                                                                                                                                  
  - Получает понятное сообщение при отказе в доступе                               
                                                                                                                                                                                                                                                                                                  
  Разработчик приложения:                                                          
  - Оборачивает UI в Guard виджеты (PermissionGuard, RoleGuard)                                                                                                                                                                                                                                   
  - Вызывает проверку прав перед критическими операциями                                                                                                                                                                                                                                          
  - Использует batch проверку для оптимизации UI                                                                                                                                                                                                                                                  
  - Подписывается на изменения прав через Riverpod

  3. Access Decision Process - Принятие решения                                                                                                                                                                                                                                                   
                                                                                                                                                                                                                                                                                                  
  3.1 Алгоритм проверки прав                                                                                                                                                                                                                                                                      
                                                                                                                                                                                                                                                                                                  
  Шаг 1: Cache Lookup                                                                                                                                                                                                                                                                             
  - Ключ: {userId}:{permission}                                                                                                                                                                                                                                                                   
  - Проверка TTL (5 минут)                                                                                                                                                                                                                                                                        
  - Hit → немедленный возврат                                                                                                                                                                                                                                                                     
  - Miss → полная проверка                                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                                  
  Шаг 2: Role Collection                                                           
  - Запрос к rbac_user_roles коллекции                                                                                                                                                                                                                                                            
  - Фильтр: userId = X AND (expiresAt IS NULL OR expiresAt > now)                                                                                                                                                                                                                                 
  - Результат: список активных ролей пользователя                                                                                                                                                                                                                                                 
                                                                                                                                                                                                                                                                                                  
  Шаг 3: Permission Aggregation (рекурсивная)                                      
  Set<String> permissions = {};                                                                                                                                                                                                                                                                   
  Set<String> visited = {};                                                                                                                                                                                                                                                                       
                                                                                                                                                                                                                                                                                                  
  void collectPermissions(roleId, depth = 0) {                                                                                                                                                                                                                                                    
    if (visited.contains(roleId) || depth > 5) return;                             
    visited.add(roleId);                                                                                                                                                                                                                                                                          
                                                                                                                                                                                                                                                                                                  
    role = getRole(roleId);                                                                                                                                                                                                                                                                       
    permissions.addAll(role.permissions);                                                                                                                                                                                                                                                         
                                                                                                                                                                                                                                                                                                  
    for (parentId in role.inheritsFrom) {                                                                                                                                                                                                                                                         
      collectPermissions(parentId, depth + 1);                                                                                                                                                                                                                                                    
    }                                                                                                                                                                                                                                                                                             
  }                                                                                                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                                                  
  Шаг 4: Wildcard Matching                                                                                                                                                                                                                                                                        
  Requested: "projects:delete:own"                                                 
  Available: ["projects:*:own", "files:read:*"]                                                                                                                                                                                                                                                   
                                                                                   
  Match logic:                                                                                                                                                                                                                                                                                    
    "projects" matches "projects" ✅                                               
    "delete" matches "*" ✅                  
    "own" matches "own" ✅                   
    → GRANTED                                                                                                                                                                                                                                                                                     
                                                         
  Шаг 5: Policy Evaluation                                                                                                                                                                                                                                                                        
  - Получить все enabled политики                                                                                                                                                                                                                                                                 
  - Сортировать по priority (DESC)                              
  - Для каждой политики:                                                                                                                                                                                                                                                                          
    - Проверить ВСЕ условия (AND логика)                                                                                                                                                                                                                                                          
    - Если все условия true:                                                                                                                                                                                                                                                                      
        - Effect = DENY → немедленный отказ                                                                                                                                                                                                                                                       
      - Effect = ALLOW → продолжить                                                                                                                                                                                                                                                               
  - Если ни одна DENY политика не сработала → разрешить                                                                                                                                                                                                                                           
                                                                                                                                                                                                                                                                                                  
  Шаг 6: Decision Creation                                                                                                                                                                                                                                                                        
  - Собрать результат с метаданными                                                                                                                                                                                                                                                               
  - Указать причину (для аудита)                                                                                                                                                                                                                                                                  
  - Перечислить примененные политики                                                                                                                                                                                                                                                              
  - Перечислить эффективные права

  4. State Management - Управление состоянием                                                                                                                                                                                                                                                     
                                                                                                                                                                                                                                                                                                  
  4.1 Кэширование                                                                                                                                                                                                                                                                                 
                                                                                                                                                                                                                                                                                                  
  AccessCache:                                                                                                                                                                                                                                                                                    
  - In-memory Map<String, CachedDecision>                                                                                                                                                                                                                                                         
  - TTL: 5 минут (hardcoded)                                                                                                                                                                                                                                                                      
  - Max size: 10,000 записей                                                                                                                                                                                                                                                                      
  - Eviction: LRU (удаление 10% старейших при переполнении)                                                                                                                                                                                                                                       
                                                                                                                                                                                                                                                                                                  
  Инвалидация:                                                                                                                                                                                                                                                                                    
  - При изменении роли → invalidateAllCache()                                                                                                                                                                                                                                                     
  - При назначении/отзыве роли → invalidateUserCache(userId)                                                                                                                                                                                                                                      
  - При изменении политики → invalidateAllCache()                                                                                                                                                                                                                                                 
                                                                                                                                                                                                                                                                                                  
  Почему кэш критичен:                                                             
  - Проверка прав происходит на КАЖДЫЙ UI элемент                                                                                                                                                                                                                                                 
  - Без кэша: 100+ запросов к БД на одну страницу                                                                                                                                                                                                                                                 
  - С кэшем: 1-2 запроса при первой загрузке

  5.2 Repository Pattern                                                                                                                                                                                                                                                                          
                                             
  VaultRoleRepository:                                                                                                                                                                                                                                                                            
  class VaultRoleRepository implements RoleRepository {                            
    final VaultStorage vault;                                                                                                                                                                                                                                                                     
                                                                                                                                                                                                                                                                                                  
    Future<AqRole?> getRole(String roleId) async {                                                                                                                                                                                                                                                
      final data = await vault.findById('rbac_roles', roleId);                                                                                                                                                                                                                                    
      return data != null ? AqRole.fromJson(data) : null;                                                                                                                                                                                                                                         
    }                                                                              
                                                                                                                                                                                                                                                                                                  
    Future<void> saveRole(AqRole role) async {                                                                                                                                                                                                                                                    
      await vault.save('rbac_roles', role.id, role.toJson());                                                                                                                                                                                                                                     
    }                                                                                                                                                                                                                                                                                             
  }

  // Получить роли пользователя

     - scope: "own"                                                                                                                                                                                                                                                                               
     - context: { ip, mfaVerified, resourceState }                                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                                                  
  5. [RBACRouter] _checkAccess handler

  6. [RBACService] can() method                                                                                                                                                                                                                                                                   
     - Засекает время начала

  7. [AccessControlEngine] canAsync()                                                                                                                                                                                                                                                             
     a) Проверяет кэш                                                                                                                                                                                                                                                                             
        - Ключ: "user_123:projects:delete:own"                                                                                                                                                                                                                                                    
        - Miss → продолжить                                                                                                                                                                                                                                                                       
                                                                                                                                                                                                                                                                                                  
     b) Получает роли пользователя                                                                                                                                                                                                                                                                
        - userRoleRepository.getUserRoles("user_123")                                                                                                                                                                                                                                             
        - Vault query: rbac_user_roles WHERE userId = 'user_123'                                                                                                                                                                                                                                  
        - PostgreSQL: SELECT data FROM rbac_user_roles WHERE ...                                                                                                                                                                                                                                  
        - Результат: [Editor, Viewer]                                                                                                                                                                                                                                                             
                                                                                   
     c) Собирает эффективные права                                                                                                                                                                                                                                                                
        - Рекурсивно обходит иерархию                                                                                                                                                                                                                                                             
        - Editor → permissions: ["projects:*:own", "files:write:own"]                                                                                                                                                                                                                             
        - Editor inherits Viewer → permissions: ["files:read:*"]                                                                                                                                                                                                                                  
        - Итого: ["projects:*:own", "files:write:own", "files:read:*"]                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                                                  
     d) Проверяет wildcard match                                                                                                                                                                                                                                                                  
        - Requested: "projects:delete:own"                                                                                                                                                                                                                                                        
        - Available: "projects:*:own"                                                                                                                                                                                                                                                             
        - Match: ✅                                                                                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                                                  
     e) Оценивает политики                                                                                                                                                                                                                                                                        
        - policyRepository.getEnabledPolicies()                                                                                                                                                                                                                                                   
        - Vault query: rbac_policies WHERE enabled = true                                                                                                                                                                                                                                         
        - Политика 1: "No delete on weekends"                                                                                                                                                                                                                                                     
          - Condition: time (Mon-Fri)                                                                                                                                                                                                                                                             
          - Today: Monday ✅                                                                                                                                                                                                                                                                      
        - Политика 2: "Require MFA"                                                                                                                                                                                                                                                               
          - Condition: mfa (required=true)                                                                                                                                                                                                                                                        
          - context.mfaVerified: true ✅                                                                                                                                                                                                                                                          
        - Все политики пройдены                                                                                                                                                                                                                                                                   
                                                                                                                                                                                                                                                                                                  
     f) Создает решение                                                                                                                                                                                                                                                                           
        - AccessDecision(allowed: true, reason: "Access granted")                                                                                                                                                                                                                                 
                                                                                                                                                                                                                                                                                                  
     g) Кэширует                                                                                                                                                                                                                                                                                  
        - cache.set("user_123:projects:delete:own", decision, ttl: 5min)
