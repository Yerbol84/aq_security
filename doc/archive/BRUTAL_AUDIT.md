- ❌ Нет backup стратегии

- ❌ Нет foreign keys (можно удалить user, но роли останутся)

**Вероятность:** 80% при первом инциденте
**Влияние:** Критичное (потеря всех пользователей)

### 2. Security breach (HIGH RISK)

**Проблемы:**
- ❌ Нет rate limiting (можно bruteforce)
- ❌ Нет CORS (можно украсть токены)
- ❌ Нет security headers (XSS, clickjacking)
- ❌ JWT secret в plaintext в .env

**Вероятность:** 60% в первый месяц
**Влияние:** Критичное (утечка данных)

### 3. Downtime (MEDIUM RISK)

**Проблемы:**
- ❌ Нет health checks в production
- ❌ Нет auto-restart
- ❌ Нет load balancing
- ❌ Нет failover

**Вероятность:** 40% в первую неделю
**Влияние:** Высокое (сервис недоступен)

### 4. Performance degradation (MEDIUM RISK)
